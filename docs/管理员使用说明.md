# 管理员使用说明

本文面向 Laylight Agent 管理员，用于日常维护个人知识库、发布访客可问答的 Agent，并查看 HR/访客根据岗位留下的评论建议。

## 1. 登录入口

1. 打开网站首页。
2. 点击右上角“管理员登录”。
3. 输入服务器 `.env` 中配置的 `ADMIN_USERNAME` 和 `ADMIN_PASSWORD`。
4. 登录成功后进入管理端。

管理员账号、密码和访客密钥只能配置在服务器环境变量或 `.env` 中，不得写入 Git、文档示例或前端代码。

## 2. 知识库管理

管理端“知识库”页用于维护访客问答背后的个人资料。

- 新建知识库：点击“新建知识库”，输入名称。
- 发布/取消发布：选择知识库后点击“发布 Agent”或“取消发布”。同一管理员同时只能发布一个知识库；发布新的知识库会自动取消旧知识库。
- 导入文件：可一次选择最多 20 个 PDF、Markdown 或 TXT。系统逐个返回成功或失败结果，成功文件分别创建后台入库任务。
- 同步网页：填写公开网页 URL，系统根据内容变化增量同步。
- 同步 Git 项目：填写服务器允许范围内的项目路径，或公开 GitHub 仓库地址。支持标准 HTTPS、以 `.git` 结尾、复制到的 `/tree/分支/...`、省略协议和 GitHub SSH 地址，后端会统一为 HTTPS 仓库地址；当前不支持私有仓库授权。系统会先生成项目资料卡、项目文档摘要、配置摘要和代码结构摘要，过滤 `.git`、`.env`、依赖目录、测试、构建产物、锁文件和明显泛化学习资料。
- 清空内容：清空当前知识库的导入来源、Chunk、向量、入库任务和知识图谱，保留知识库本身，方便重新导入。

“已导入内容”按文件、网页和 Git 项目分栏展示来源、版本数、Chunk 数、状态和最近更新时间：

- 文件：勾选一个或多个文件后批量删除，也可逐行删除。
- 网页：点击“更新”从原 URL 重新抓取；内容变化时创建新版本。点击“删除”移除全部版本和检索内容。
- Git 项目：点击“更新”从原仓库重新克隆并清洗；Commit 或清洗器版本变化时创建新版本。点击“删除”移除全部版本和检索内容。
- 删除会清理关联 Chunk、向量、任务、引用证据和磁盘文件；来源正在入库时需等待任务结束。

上传或同步后，前端会显示后台任务进度。任务成功后，Embedding 和活动版本切换已完成，问答检索可以使用；知识图谱候选抽取是增强步骤，可能只处理部分 Chunk 或因模型结构化输出失败而跳过，不影响用户端先进行问答测试。

## 3. 候选知识审核

入库任务会从 Chunk 中抽取候选实体和关系，并绑定来源证据。

- “通过”：候选知识进入可用状态，并记录修订。
- “拒绝”：候选知识不会进入有效图谱。
- 审核时优先检查实体名称、关系方向和证据是否来自真实原文，不要只看模型置信度。

当前知识图谱用于知识沉淀、可视化和学习复习，问答主链路仍以 pgvector RAG 检索为主。管理员端“图谱”页会显示候选/已通过实体和关系的基础可视化，候选知识仍需审核后再作为可信事实沉淀。

访客端左侧栏提供留言建议区。访客可填写岗位和评论，也能看到其他访客留下的岗位与对应建议；管理员仍可在“评论”页查看完整列表。

## 4. 学习复习

“复习”页展示系统根据实体生成的学习复习任务。

- 评分 1 到 5 表示本次掌握情况。
- 分数越高，下次复习间隔越长；分数低会缩短间隔。
- 该功能用于辅助管理员持续学习，不会直接向访客暴露。

## 5. HR 评论区

访客对话界面底部提供“岗位”和“评论”两个输入框，并展示提示：

```text
欢迎根据岗位对我提出学习与补充建议，谢谢:D
```

管理员在“评论”页查看最新 HR/访客建议。

- 岗位：访客认为你正在匹配或需要补充的岗位方向。
- 评论：访客对简历、项目、技能栈或学习方向的建议。
- 评论与访客身份、可选会话 ID 绑定；访客删除会话后，评论仍保留，但会话引用会置空。

建议定期把高价值评论整理到学习计划或简历优化记录中。评论区不是公开留言墙，当前只提供管理员查看，不提供访客查看历史评论。

## 6. 访客端验证

每次发布新版本后，建议用访客密钥走一遍最小链路：

1. 输入 `VISITOR_ACCESS_KEY` 进入网站。
2. 新建对话并发送一个与已发布知识库相关的问题。
3. 确认回答为流式输出，并能看到引用证据。
4. 删除当前对话，确认不会影响其他访客。
5. 提交一条岗位评论，再用管理员登录查看评论页。

如果访客端提示 Agent 不可用，通常是没有已发布知识库，或发布的知识库尚未完成入库。

## 7. 上线安全

- 生产环境必须替换 `JWT_SECRET`、管理员密码、访客密钥、DeepSeek Key 和 Embedding Token。
- 公开仓库不得包含真实 `.env`、API Key、Token、简历正文和上传文件。
- Nginx 必须代理 `/api/` 到后端，并对 SSE 路径关闭缓冲。
- 数据库和 Redis 不对公网开放。
- 迁移执行顺序为：备份数据库 -> 拉取镜像/代码 -> `alembic upgrade head` -> 重启服务 -> 验证访客和管理员链路。

## 8. 常见问题

### 访客输入密钥后进不去

检查 `VISITOR_ACCESS_KEY` 是否配置在后端运行环境中，前端不会直接校验密钥。

### 管理员登录失败

检查 `ADMIN_USERNAME`、`ADMIN_PASSWORD` 是否与服务器环境变量一致，并确认后端已重启加载新配置。

### 评论页为空

先用访客端提交一条评论；若接口成功但页面不显示，检查管理员 Token 是否有效以及 `/api/v1/conversations/feedback` 是否被 Nginx 正确代理。

### 问答没有流式输出

检查 Nginx SSE 配置是否关闭代理缓冲，并确认路径 `/api/v1/conversations/{id}/messages:stream` 转发到后端。

### GitHub 仓库同步返回 422

确认地址至少包含 GitHub 用户名和仓库名，并且仓库可公开访问。系统会区分地址格式错误、仓库不存在或为私有仓库、GitHub 网络不可达和克隆超时；不要把访问令牌拼接到仓库 URL。

### 回答与已导入简历或项目不符

先确认只有目标知识库显示为“已发布”，并在“已导入内容”中确认简历、网页和 Git 项目状态为“可用于问答”。项目或网页内容修改后点击“更新”。没有有效证据时系统应明确说明资料不足；若仍出现资料中不存在的项目名称，保留问题和来源列表用于排查，不要通过放宽 Prompt 让模型猜测。
