# 项目 1 技术八股全解：索引与学习方法

## 项目定位

项目 1 是基于 Python 的持续学习型个人知识库与知识图谱 Agent。核心链路为：知识源增量同步 -> 文档解析与向量存储 -> 实体关系抽取与证据绑定 -> 候选审核和冲突处理 -> RAG/图谱联合检索 -> Agent 回答与引用 -> 学习事件和复习计划。

这些文档不是脱离项目背概念。每项技术都按六个问题学习：

1. 为什么项目需要它？
2. 它解决了什么问题，有什么收益和代价？
3. 在项目哪个模块、哪条链路中使用？
4. 底层原理是什么？
5. 容易出现什么故障，如何定位和解决？
6. 面试官可能如何继续追问？

## 推荐学习顺序

先完整讲通一次业务链路，再按 `01 -> 02 -> 03 -> 04 -> 05` 学习。每学完一篇至少完成三个输出：

- 用自己的话在 2 分钟内回答“为什么选它而不是替代方案”。
- 在项目中找到或实现对应代码，并能指出入口、状态变化和异常路径。
- 为至少一个故障写测试，例如模型超时、重复同步、Redis 不可用、向量库过滤错误。

## 项目统一回答框架

面试介绍技术方案时使用：

```text
背景问题 -> 选择方案 -> 核心实现 -> 可靠性处理 -> 测试指标 -> 局限和演进
```

示例：为避免知识库网页每次全量向量化，优先使用 ETag/Last-Modified 判断变化，无法获得时计算规范化正文哈希；变化后生成新版本 Chunk，全部写入成功再切换当前版本，失败继续提供旧版本。这样减少 Embedding 调用并避免半更新，但需要维护版本与清理任务。

## 掌握标准

- 初级：知道名词，能描述用途。
- 可投递：能结合项目说明选型、代码路径、异常处理和测试结果。
- 面试优秀：能比较替代方案、解释底层原理、指出当前实现边界及规模扩大后的演进。

只有达到“可投递”才写进简历；只有本人实测的数据才写成量化结果。
