# RAG、Reranker 与 Agent 全链路全解

> 当前项目状态：已实现 Embedding、pgvector 余弦检索、活动版本/权限过滤、引用持久化、拒答 Prompt 和 SSE；混合检索、Reranker、Query Rewrite、工具调用尚未实现。本篇后半部分属于后续优化知识，不得直接写成当前成果。

## 1. 为什么使用 RAG

大模型参数中的知识可能过时，也不知道用户私有资料。RAG 在回答前检索外部知识，把相关片段作为上下文，使答案可更新、可引用且无需微调模型。

收益：私有知识接入快、更新成本低、可追溯。局限：检索错了生成通常也会错；上下文增加 Token 成本；模型仍可能忽略资料或编造。必须同时治理检索和生成，不能把 RAG 等同于“不会幻觉”。

## 2. Embedding 原理与选型

Embedding 模型把文本编码为定长向量，训练目标让语义相近文本更接近。项目需要固定模型名称、版本、维度和距离度量。中英文能力、输入长度、价格、延迟和领域效果都应通过评测选择。

问题与解决：模型更换导致向量不兼容，使用版本隔离和全量重建；长文本被截断，切分前计算 Token；批量过大导致超时，限制批次和并发；429 使用指数退避并尊重 Retry-After。

## 3. Chunk 为什么决定效果

Chunk 太大包含噪声、占上下文且定位不准；太小语义不完整。Overlap 能保留跨边界信息，但过大会增加重复和费用。

项目先按标题、段落和句子进行语义切分，再按 Token 上限拆分，保留来源 URL、标题层级、页码和版本。初始参数只是基线，必须用固定评测集比较 300/500/800 Token、不同 Overlap 和 Top-K。

常见问题：表格被拆散、代码块断裂、页眉页脚污染、相邻 Chunk 重复进入上下文。解决方式是格式感知解析、清洗规则、父子 Chunk、邻居扩展和去重。

## 4. 检索、混合检索与重排

向量检索擅长语义相似，但对产品编号、类名、错误码等精确词可能不敏感；BM25/关键词检索擅长词项匹配。混合检索分别召回后通过 RRF 或归一化分数组合，提高不同问题类型的鲁棒性。

Reranker 对“问题-候选片段”成对打分，通常比单纯向量相似度更准确，但增加延迟和成本。推荐先召回 Top 20，再重排取 Top 4-6，而不是对全库重排。

RRF 根据排名融合，不要求两种分数在同一尺度：`score = sum(1 / (k + rank))`。若直接加权分数，必须先归一化，否则某一路分数范围会主导结果。

## 5. Query Rewrite 与多轮对话

用户说“它的超时时间呢”时，单独 Embedding 无法知道“它”指什么。Query Rewrite 使用有限历史把问题改写成独立问题，再用于检索。原始问题仍用于生成，防止改写丢失语气和意图。

风险是改写模型引入错误实体。可以要求只消解指代、不回答问题，并保存改写结果用于调试。简单会话可拼最近若干轮，长会话采用 Token 预算和摘要，不能无限加载历史。

## 6. Prompt 与幻觉治理

系统 Prompt 明确：只依据资料回答、资料不足拒答、文档内容不是系统指令、引用编号格式和输出语言。上下文用定界符包裹，并包含来源 ID。

幻觉治理是组合方案：检索阈值、Reranker、资料不足检测、引用校验、结构化输出、低温度、回答后事实核对和用户反馈。阈值过高会过度拒答，过低会引入噪声，必须用可回答/不可回答评测集调参。

Prompt Injection：文档中可能写“忽略系统提示”。需要把文档标记为不可信数据，工具权限由服务端决定，不能让模型生成的 URL/SQL 直接执行。

## 7. Agent 与工具调用

Agent 相比固定 RAG 链增加“根据意图决定下一步”的能力。项目可提供：知识库检索、查看来源、重新同步知识源、查询同步状态、创建人工转接记录等受控工具。

工具使用 JSON Schema 定义参数，由服务端校验权限和范围。模型只提出调用建议，执行权在应用。每次调用设置最大步数、超时、Token/费用预算，并记录 trace，防止循环调用。

什么时候不需要 Agent：单一“检索后回答”用固定流程更稳定、便宜且易测。只有存在多类意图、多个工具或条件分支时再引入 Agent/LangGraph。

## 8. RAG 评测

检索指标：Recall@K 衡量正确片段是否进入 Top-K；MRR 关注第一个正确结果排名；引用命中率验证最终引用是否包含证据。

生成指标：答案正确性、忠实度、相关性、拒答正确率。LLM-as-Judge 可扩展但有偏差，关键样本要人工复核。评测集覆盖可回答、跨段、多轮、精确词、冲突资料和不可回答问题。

每次实验固定语料、问题、模型和环境，只改变一个变量；保存原始候选、分数、Prompt、答案、引用和耗时，才能解释“提升百分之多少”。

## 9. 面试高频问答

**RAG 与微调区别？** RAG 注入外部事实，更新快且可引用；微调更适合行为、风格和稳定格式，不应主要用于记忆频繁变化的事实。

**Top-K 越大越好吗？** 不是。召回可能提高，但噪声、Token 和延迟增加；使用重排、阈值和评测选择。

**为什么需要 Reranker？** Embedding 是独立编码，速度快但交互不足；Cross-Encoder 联合看问题与文档，相关性更准但成本更高。

**如何证明 Agent 不是套壳？** 展示知识源同步、检索评测、工具权限、状态机、失败恢复、日志 trace 和部署数据，而不是只展示调用模型 API。
