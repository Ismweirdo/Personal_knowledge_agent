# 上线运行手册

## 发布前

- 使用随机强 `JWT_SECRET`，配置数据库、Redis、DeepSeek 与 Embedding 密钥。
- 设置 `APP_ENV=production` 和 `RATE_LIMIT_ENABLED=true`。
- 运行 Alembic 迁移，并通过交互式 CLI 创建唯一管理员。
- 只开放 80/443；PostgreSQL、Redis 和 `/metrics` 不对公网开放。
- 确认私人上传目录、`.env`、备份目录不在 Git 仓库中。

## 验证

1. `/health` 返回 200，`/health/ready` 的数据库和 Redis 均为 up。
2. 普通用户访问知识写接口返回 403。
3. 上传、索引、发布、访客 SSE 问答和引用链路可用。
4. 登录、上传和问答超过限额后返回 429 与 `Retry-After`。
5. 容器重启后数据库、上传文件和索引仍存在。

## 备份恢复

每日执行 `deploy/backup-postgres.sh`，默认保留 14 天。上传文件卷需要使用服务器快照或对象存储版本控制另行备份。每月至少在隔离环境执行一次 `restore-postgres.sh`，并验证管理员登录、知识库数量、Chunk 数和问答引用。

## 故障处理

- 按 requestId 查询结构化请求日志，不记录 Token、Prompt 或文档正文。
- 就绪检查失败时先检查数据库连接、Redis 和迁移版本。
- 模型服务失败时保留旧知识版本，禁止删除现有活动向量后重试。
- 回滚应用镜像时确认其支持当前数据库迁移版本；数据库回滚必须先备份。
